Site Plan

Reason of the name

I chose the name "Gemini" for my e-portfolio because it represents two sides of thinking and combines two things I like most in the world: web development and communication. By putting them together, I created a name that represents both me and my work.

Site Purpose

The purpose of this site is to catch the attention of potential recruiters who are interested in a web developer with good organizational skills, best programming practices, and the ability to apply concepts effectively. It also focuses on both client and user experience aspects of the career.

Scenarios

Scenario #1

Visitor: A potential recruiter from a tech company.
Question: "How does this web developer ensure that best practices are followed in their projects?"
Content Focus: Highlighting examples of projects where best practices in coding,
such as clean code principles, version control, and testing, were implemented. Including case studies or testimonials from previous clients or employers can also be helpful.

Scenario #2

Visitor: A client looking for a web developer to enhance their website's user experience.
Question: "What strategies does this developer use to improve client and user experience on a website?"
Content Focus: Showcasing a portfolio of previous work that emphasizes user interface (UI) and user experience (UX) design improvements. Discussing methods like user research, usability testing, and responsive design can illustrate your expertise in this area.
